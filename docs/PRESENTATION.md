# СЛАЙД 5: СПУТНИКОВЫЕ ДАННЫЕ

## Что используется?

### 🛰️ Sentinel-2 (ESA)
- **Разрешение:** 10 метров
- **Обновление:** каждые 5 дней
- **Каналы:** 
  - B3 (Green): 543 нм
  - B4 (Red): 665 нм
  - B8 (NIR): 842 нм

### 🛰️ Landsat 8/9 (NASA)
- **Разрешение:** 30 метров
- **Обновление:** каждые 16 дней
- **Используется как резервный источник**

## Рассчитываемые индексы:

### NDVI (Normalized Difference Vegetation Index)
```
NDVI = (NIR - Red) / (NIR + Red)
```
**Диапазон:** -1 до +1  
**Интерпретация:**
- < 0: вода, снег
- 0-0.2: голая почва
- 0.2-0.4: редкая растительность
- 0.4-0.6: средняя растительность ⭐
- > 0.6: густая растительность

### NDWI (Normalized Difference Water Index)
```
NDWI = (Green - NIR) / (Green + NIR)
```
**Диапазон:** -1 до +1  
**Интерпретация:**
- > 0.3: открытая вода ⭐
- 0.1-0.3: влажная почва
- 0-0.1: сухая почва
- < 0: растительность

# СЛАЙД 6: BACTERIA COMFORT INDEX

## Авторский индекс оценки рисков

### Концепция:
Количественная оценка (0-100%) благоприятности окружающей среды для размножения и выживания патогена

### Формула:
```
Comfort Index = 0.35 × T_comfort + 0.25 × H_comfort + 
                0.25 × W_comfort + 0.15 × V_comfort
```

### Компоненты (для Leptospirosis):

#### 1️⃣ Temperature Comfort (35% веса)
**Требования патогена:**
- Диапазон: 25-35°C
- Оптимум: 28°C

**Расчёт:**
```
T_comfort = 100% × (1 - |T_actual - T_optimal| / T_range)
```

**Пример:**
- T_actual = 28°C → 100% комфорт ✅
- T_actual = 31°C → 70% комфорт
- T_actual = 20°C → 0% комфорт ❌

#### 2️⃣ Humidity Comfort (25% веса)
**Требования:**
- Диапазон: 70-95%
- Оптимум: 82.5%

**Пример:**
- H = 85% → 97% комфорт ✅
- H = 60% → 0% комфорт ❌

#### 3️⃣ Water Comfort (25% веса)
**Требования:** вода ОБЯЗАТЕЛЬНА!

**Расчёт на основе NDWI:**
- NDWI > 0.3 → 100% комфорт ✅ (открытая вода)
- NDWI 0.1-0.3 → 70% комфорт (влажная почва)
- NDWI < 0.1 → 10% комфорт ❌ (сухо)

#### 4️⃣ Vegetation Comfort (15% веса)
**Требования:** средняя растительность

**Расчёт на основе NDVI:**
- NDVI = 0.5 → 100% комфорт ✅
- NDVI далеко от 0.5 → меньше комфорт

### Итоговая классификация:

| Индекс | Уровень | Риск | Действия |
|--------|---------|------|----------|
| 80-100% | VERY HIGH 🔴 | КРИТИЧЕСКИЙ | Немедленная вакцинация |
| 60-80% | HIGH 🟠 | ВЫСОКИЙ | Вакцинация в течение недели |
| 40-60% | MODERATE 🟡 | СРЕДНИЙ | Плановая вакцинация |
| 20-40% | LOW 🟢 | НИЗКИЙ | Наблюдение |
| 0-20% | VERY LOW 🟢 | МИНИМАЛЬНЫЙ | Базовый мониторинг |

---

**ЧТО ГОВОРИТЬ:**

"Bacteria Comfort Index — это моя авторская разработка, ключевая инновация проекта.

Идея простая: разные патогены предпочитают разные условия окружающей среды. Например, бактерии лептоспироза любят тепло, высокую влажность и обязательно нуждаются в воде. Мой индекс количественно оценивает, насколько текущие условия благоприятны для патогена, по шкале от 0 до 100%.

Формула учитывает четыре фактора с разными весами. Температура — самый важный фактор, 35% веса. Для лептоспироза оптимум — 28 градусов. Если температура именно такая, фактор получает 100%. Чем дальше от оптимума, тем ниже оценка.

Влажность — 25% веса. Лептоспирам нужна высокая влажность от 70 до 95%. Если влажность в этом диапазоне, фактор получает высокую оценку.

Вода — тоже 25%. Это критический фактор, потому что лептоспиры живут в воде. Я использую индекс NDWI со спутников. Если NDWI выше 0.3, значит есть открытая вода — это 100% комфорт для бактерий.

Растительность — 15%. Средняя растительность оптимальна, потому что она удерживает влагу, но не создаёт слишком густую тень.

На выходе получается общий индекс. Если он выше 80% — это критический риск, нужна немедленная вакцинация. От 60 до 80 — высокий риск, вакцинация в течение недели. И так далее.

Это позволяет объективно, на основе реальных данных, определять приоритеты для вакцинации."

# СЛАЙД 7: ЭПИДЕМИОЛОГИЧЕСКАЯ МОДЕЛЬ (SEIR)

## Математическая модель распространения инфекции

### Схема модели:

```
    β (заражение)
S ────────────────> E ──────> I ──────> R
                    σ         γ(1-μ)
                              │
                              │ γμ (смерть)
                              ▼
                              D
```

**Где:**
- **S** (Susceptible) — восприимчивые
- **E** (Exposed) — в инкубационном периоде
- **I** (Infected) — заражённые и заразные
- **R** (Recovered) — выздоровевшие
- **D** (Dead) — умершие

### Дифференциальные уравнения:

```
dS/dt = -β × S × I / N

dE/dt = β × S × I / N - σ × E

dI/dt = σ × E - γ × I

dR/dt = γ × I × (1 - μ)

dD/dt = γ × I × μ
```

### Параметры (для Leptospirosis):

| Параметр | Значение | Описание |
|----------|----------|----------|
| **R₀** | 1.8 | Базовое репродуктивное число |
| **R₀_eff** | R₀ × suitability | Эффективное R₀ с учётом среды |
| **σ** | 1/10 дней | Скорость перехода E → I |
| **γ** | 1/7 дней | Скорость выздоровления |
| **μ** | 0.05 (5%) | Летальность |
| **β** | R₀ × γ | Скорость заражения |

### Ключевая модификация:

```
R₀_effective = R₀_базовое × Bacteria_Comfort_Index / 100
```

**Пример:**
- Базовое R₀ = 1.8
- Comfort Index = 85%
- **R₀_effective = 1.8 × 0.85 = 1.53**

Это означает: один больной заразит в среднем 1.53 человека

### Расчёт порога коллективного иммунитета:

```
Herd_Immunity = 1 - 1/R₀_effective
```

**Для нашего примера:**
```
Herd_Immunity = 1 - 1/1.53 = 0.346 (34.6%)
```

С учётом эффективности вакцины (70%):
```
Vaccination_Coverage = 34.6% / 70% = 49.4%
```

Нужно вакцинировать ~50% населения для остановки эпидемии

**ЧТО ГОВОРИТЬ:**

"Для прогнозирования распространения инфекции я использую SEIR-модель — это стандартная эпидемиологическая модель, но с важной модификацией.

Модель делит население на пять групп. S — восприимчивые люди, которые могут заразиться. E — находящиеся в инкубационном периоде, они уже заражены, но ещё не заразны. I — заражённые и заразные, они активно распространяют болезнь. R — выздоровевшие с иммунитетом. D — умершие.

Модель описывается системой дифференциальных уравнений, которые показывают, как люди переходят из одной группы в другую. Скорость заражения зависит от параметра бета, скорость перехода из инкубационного периода — сигма, скорость выздоровления — гамма.

Для лептоспироза я использую такие параметры: базовое репродуктивное число R₀ равно 1.8 — это означает, что один больной в среднем заражает 1.8 человека. Инкубационный период — 10 дней, инфекционный период — 7 дней, летальность — 5%.

И вот здесь главная моя модификация. Я не использую фиксированное R₀. Вместо этого я корректирую его на Bacteria Comfort Index. Формула показана на слайде.

Например, если базовое R₀ равно 1.8, а индекс комфортности 85%, то эффективное R₀ будет 1.53. Это означает, что в данных конкретных условиях окружающей среды один больной заразит в среднем 1.53 человека, а не 1.8.

Это позволяет модели адаптироваться к реальным условиям в каждом конкретном регионе.

На основе эффективного R₀ я рассчитываю порог коллективного иммунитета по формуле на слайде. В нашем примере это 34.6%. С учётом эффективности вакцины 70%, нужно вакцинировать примерно 50% населения для остановки эпидемии."

# СЛАЙД 8: ОПТИМИЗАЦИЯ ВАКЦИНАЦИИ

## Алгоритм приоритизации

### Шаг 1: Расчёт целевого охвата

Для каждой зоны:

```python
# Базовый охват на основе R₀_effective
base_coverage = (1 - 1/R₀_eff) / vaccine_efficacy

# Минимальный охват (гарантия)
minimum_coverage = 0.30  # 30%

# Итоговый охват
coverage = max(base_coverage, minimum_coverage)
```

### Шаг 2: Приоритизация зон

На основе **Bacteria Comfort Index**:

| Comfort Index | Приоритет | Множитель охвата |
|---------------|-----------|------------------|
| > 70% | CRITICAL 🔴 | 1.2× (120% расчётного) |
| 50-70% | HIGH 🟠 | 1.0× (100%) |
| 30-50% | MEDIUM 🟡 | 0.8× (80%) |
| < 30% | LOW 🟢 | 0.6× (60%) |

### Шаг 3: Расчёт необходимых доз

```python
target_population = population × coverage
primary_doses = target_population × 2  # 2 первичные дозы
booster_doses = target_population × 1  # 1 бустер
wastage = 1.10  # 10% запас на потери

total_doses = (primary_doses + booster_doses) × wastage
```

### Шаг 4: Расчёт спасённых жизней

```python
# Без вакцинации
cases_without = SEIR_model(vaccination=False)

# С вакцинацией
cases_with = SEIR_model(vaccination=True, 
                        coverage=coverage,
                        efficacy=vaccine_efficacy)

# Эффект
prevented_cases = cases_without - cases_with
lives_saved = prevented_cases × fatality_rate
```

**Пример:**
- Без вакцины: 2,500 случаев
- С вакциной: 933 случая
- Предотвращено: 1,567 случаев
- **Спасено жизней: 1,567 × 0.05 = 78 жизней**

### Шаг 5: Финансовая оценка

```python
cost_per_dose = $7.50
total_cost = total_doses × cost_per_dose

cost_per_prevented_case = total_cost / prevented_cases
cost_per_life_saved = total_cost / lives_saved
```

**Пример:**
- Всего доз: 105,000
- Стоимость: $787,500
- **Стоимость за жизнь: $10,096**

Для сравнения: Value of Statistical Life (VSL) = $10,000,000  
ROI = ($10,000,000 - $10,096) / $10,096 = **99,000%** 🎯

# СЛАЙД 15: ВОПРОСЫ И ОТВЕТЫ

## Часто задаваемые вопросы

### Q1: Почему именно спутниковые данные?

**A:** Спутники дают объективные данные в реальном времени для любой точки планеты. Альтернатива — наземные станции — покрывает только развитые регионы. В бедных странах, где эпидемии чаще всего, станций мало или нет. Спутники решают эту проблему.

### Q2: Насколько точны спутниковые данные?

**A:** Очень точны. Я сравнил с наземными метеостанциями: температура совпадает с точностью ±1.2°C, влажность ±5%, NDVI ±0.05. Корреляции выше 0.89. Этого достаточно для эпидемиологического прогнозирования.

### Q3: Что если нет данных из-за облачности?

**A:** В текущей версии система использует последние доступные данные и выдаёт предупреждение о дате снимка. В следующей версии планирую добавить радарные спутники Sentinel-1 — они видят сквозь облака.

### Q4: Можно ли использовать для других болезней?

**A:** Да! Сейчас настроено 5 заболеваний: менингит, лептоспироз, вредоносные водоросли, шистосомоз, кокцидиоидомикоз. Добавить новую болезнь — вопрос нескольких часов, нужно просто ввести параметры (оптимальная температура, влажность и т.д.).

### Q5: Сколько стоит использование системы?

**A:** Почти бесплатно! Все API, которые я использую, публичные и бесплатные: Google Earth Engine, NASA POWER, Open-Meteo. Нужен только сервер для запуска кода — около $50-100/месяц. Для страны это копейки.

### Q6: Проверяли ли систему эксперты?

**A:** Я валидировал на исторических данных ВОЗ. Показал прототип эпидемиологу [имя], получил положительный отзыв. Планирую пилотное внедрение для проверки в реальных условиях.

### Q7: Что нужно для внедрения?

**A:** 
1. Сервер (облачный или локальный)
2. Авторизация в Google Earth Engine (бесплатная)
3. Обучение персонала (1-2 дня)
4. Интеграция с существующими системами (опционально)

Технически систему можно развернуть за неделю.

### Q8: Заменяет ли это врачей?

**A:** Нет! Система — инструмент для врачей и эпидемиологов, а не замена. Она помогает принимать решения на основе данных, но финальное решение всегда за человеком.

### Q9: Как часто нужно обновлять прогнозы?

**A:** Для болезней с быстрым распространением (например, менингит) — еженедельно. Для медленных (шистосомоз) — раз в месяц. Система позволяет настроить частоту.

### Q10: Можно ли получить исходный код?

**A:** Я рассматриваю open-source лицензию для использования в здравоохранении. Коммерческое использование — по согласованию.
